What is it?

    It is a personal project where I built a web app to control the servo to open the locked door for me.

Strategy

      1.ESP32 Login in the same wifi as your device and be a server at certain port.

      2. Web app user interface design

      3.Web client write http request for the ESP32 and ESP32 parse the GET request and response with its GPIO.

      4.Hardware design (servo mounting position, bread board and power supply)

Results

    Below is the web app interface to control the servo:
    Press the button on the web app, the internet will unlock the door for you.

What have I learnt from this project?

    1. How to use Http request with hardware

    2. Mounting position was hard to be decided since I cannot drill a hole on the landlord's door.

    3. Not having common GND for all devices can be a potential bug.

    4. When the target position of a servo is far from the current position, the servo was vibrating but I thought the servo was not given proper signal since I was using Arduino IDE for ESP32. But choosing NodeMCU-32S in Arduino IDE is compatible with the board (ESP32) I bought from Amazon.
